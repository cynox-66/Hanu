# Hanu

## Product Charter

**Version:** 1.0
**Status:** Draft
**Owner:** Dev Jaiswal
**Last Updated:** July 2026

---

# Mission

Hanu exists to reduce the daily operational workload of home-based business owners through a calm, mobile-first business assistant.

It is not an e-commerce platform.

It is not an ERP.

It is not accounting software.

It is an operational assistant that helps business owners spend less time managing their business and more time running it.

---

# Vision

Become the simplest business operating system for home entrepreneurs.

Every interaction should save time, reduce stress, or eliminate repetitive work.

---

# Problem Statement

Thousands of home businesses in India operate almost entirely through WhatsApp, Instagram, Facebook, and offline sales.

These businesses usually manage:

- Products
- Inventory
- Customers
- Orders
- Payments
- Courier
- Marketing

using a mixture of:

- WhatsApp chats
- Phone gallery
- Memory
- Notebooks
- Manual copy-pasting

This workflow creates:

- Lost orders
- Inventory mistakes
- Repeated work
- High cognitive load
- Poor scalability
- Operational stress

Existing software solves enterprise problems.

Very little software is designed specifically for a single non-technical person running a home business from an Android phone.

Hanu exists to solve that gap.

---

# Primary User

Initial User

- One home-based boutique owner
- Uses only an Android phone
- Limited technical experience
- Runs the business alone
- Sells primarily through WhatsApp
- Lives in India

Future versions may support additional business types.

Version 1 does not.

---

# Product Philosophy

Hanu should feel like an assistant.

Not a dashboard.

Not a database.

Not an ERP.

The application should guide the user through today's work instead of exposing raw business data.

Every screen should answer:

> What should I do next?

instead of

> What data do you want to manage?

---

# Core Principles

## 1. Mobile First

Every feature is designed for phones before larger screens.

Desktop is optional.

Mobile is mandatory.

---

## 2. Camera First

Most business workflows begin with a product photo.

The camera is treated as the primary input device.

---

## 3. Offline First

The application should continue functioning without internet.

Synchronization happens automatically whenever connectivity returns.

The user should never lose work because of poor network conditions.

---

## 4. Local First

User data belongs on the user's device first.

Cloud services enhance the experience.

They should not be required for basic functionality.

---

## 5. Action First

The interface prioritizes actions.

Examples:

- Add Product
- Pack Orders
- Share Product
- Mark Sold

instead of

- Products
- Inventory
- Database

---

## 6. Calm Software

The interface should reduce stress.

Avoid:

- Information overload
- Dense dashboards
- Enterprise terminology
- Unnecessary charts

Prefer:

- White space
- Large touch targets
- Clear language
- One primary action per screen

---

## 7. Reliability First

Correct software is more valuable than feature-rich software.

A small reliable application is preferable to a large unreliable one.

---

## 8. Invisible Complexity

Complex engineering belongs behind the scenes.

The user should never need to understand:

- Databases
- Synchronization
- APIs
- Caching
- Logistics integrations

# Product Goals

Version 1 goals:

- Organize products
- Reduce repetitive WhatsApp work
- Prevent inventory mistakes
- Simplify order management
- Prepare shipments
- Reduce operational stress

Version 1 is NOT intended to:

- Replace WhatsApp
- Replace Instagram
- Replace accounting software
- Replace ERP software
- Replace marketplaces

---

# Success Metrics

Hanu is successful if it can:

- Reduce average order processing time.
- Reduce repeated customer questions.
- Reduce inventory mistakes.
- Eliminate notebook-based tracking.
- Reduce manual copy-pasting.
- Make onboarding possible in under 15 minutes.
- Be usable without external assistance.

---

# Non-Goals

Version 1 will NOT include:

- Marketplace
- Customer accounts
- Complex analytics
- Accounting
- GST management
- Discounts
- Coupons
- Loyalty programs
- Multi-user support
- Staff management
- Multi-store management

These are intentionally excluded.

---

# Product Values

Every feature must satisfy at least one of these values.

✓ Saves time

✓ Reduces stress

✓ Removes repetitive work

✓ Prevents mistakes

✓ Makes the business easier to run

If a feature satisfies none of these values, it should not exist.

---

# Design Rule

The application should never ask the user to manage the business.

The application should tell the user what needs to be done next.

---

# Engineering Rule

Whenever there is a tradeoff between:

Complexity vs Simplicity

choose Simplicity.

Whenever there is a tradeoff between:

Features vs Reliability

choose Reliability.

Whenever there is a tradeoff between:

Automation vs User Control

choose User Control.

---

# Long-Term Vision

Hanu should eventually become the trusted business companion for millions of small home-based entrepreneurs.

That future begins by solving one person's problems exceptionally well.
